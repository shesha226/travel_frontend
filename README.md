# Local Travel Experiences Platform

## Project Overview
This is a modern web application built to connect travelers with authentic, locally hosted experiences. Users can browse various travel listings, view detailed information including pricing, duration, and host details, and proceed to book their desired experiences.

## Tech Stack
* **Frontend:** Next.js (App Router), React, Tailwind CSS, Lucide Icons
* **Backend:** Node.js / Express.js 
* **Database:** MongoDB (
* **Deployment:** Vercel (Frontend) 

## Setup Instructions
To run this project locally, follow these steps:

1. Clone the repository:
   `git clone https://github.com/yourusername/your-repo-name.git`
2. Navigate into the directory:
   `cd your-repo-name`
3. Install dependencies:
   `npm install`
4. Set up environment variables:
   Create a `.env.local` file in the root directory and add:
   `NEXT_PUBLIC_API_URL=http://localhost:5000/api` (Update with your local backend URL)
5. Start the development server:
   `npm run dev`
6. Open `http://localhost:3000` in your browser.

## Features Implemented
* Dynamic home page with responsive travel experience cards.
* Detailed listing pages fetching real-time data from the backend API.
* Clean, modern, and mobile-responsive UI using Tailwind CSS.
* Loading states and graceful error handling for missing listings.
* Interactive UI elements (Save, Share, Book buttons).

## Architecture & Key Decisions
* **Technology Stack Choice:** Next.js was chosen for its excellent performance, Server-Side Rendering (SSR) capabilities (which is crucial for SEO in a travel booking platform), and seamless integration with React. Tailwind CSS was used to ensure a highly customizable and responsive design without writing custom CSS files.

## Product Thinking Question
**If this platform had 10,000 travel listings, what changes would you make to improve performance and user experience?**

If the platform scaled to 10,000 listings, the primary focus would be on optimizing data fetching and database queries. I would immediately implement server-side pagination or infinite scrolling to prevent overwhelming the frontend with massive data payloads and to reduce initial load times. On the database side, adding indexes to frequently queried fields like location, price, and category is crucial for fast lookups. To further reduce database load and latency, I would integrate a caching layer like Redis for frequently accessed listings and popular search results. Finally, I would enhance the search experience by implementing a robust search engine like Elasticsearch to support typo-tolerance, advanced filtering, and faster response times, ensuring a seamless User Experience even with a large dataset.
